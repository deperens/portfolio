风格化卡通shader

GenshinToon_Body

概述

复刻原神风格的卡通角色 Shader，包含 Face 和 Body 两个部分。核心目标是用美术可控的方式实现风格化阴影，而非物理真实的光照。
技术亮点：

- Face：SDF 距离场 + 头部方向向量实现左右自动阴影
- Body：LightMap（AO + RampID）+ 多材质 Ramp 混合 + 昼夜切换

GenshinToon_Body 核心管线

- 输入：BaseMap、LightMap（G=AO, A=RampID）、RampTex、顶点色G通道
- 关键算法：半兰伯特 + AO 混合 → Ramp UV 计算 → RampShadowID 分段采样
- 特色：支持5级 Ramp 材质切换，美术可通过 LightMap Alpha 和顶点色精细控制阴影风格

核心代码片段（RampShadowID）

float RampShadowID(float input, float useShadow2, float useShadow3, float useShadow4, float useShadow5, float shadowValue1, float shadowValue2, float shadowValue3, float shadowValue4, float shadowValue5) { // 根据input值将模型分为5个区域 float v1 = step(0.6, input) * step(input, 0.8); // 0.6-0.8区域 float v2 = step(0.4, input) * step(input, 0.6); // 0.4-0.6区域 float v3 = step(0.2, input) * step(input, 0.4); // 0.2-0.4区域 float v4 = step(input, 0.2); // 0-0.2区域 // 根据开关控制是否使用不同材质的值 float blend12 = lerp(shadowValue1, shadowValue2, useShadow2); float blend15 = lerp(shadowValue1, shadowValue5, useShadow5); float blend13 = lerp(shadowValue1, shadowValue3, useShadow3); float blend14 = lerp(shadowValue1, shadowValue4, useShadow4); // 根据区域选择对应的材质值 float result = blend12; // 默认使用材质1或2 result = lerp(result, blend15, v1); // 0.6-0.8区域使用材质5 result = lerp(result, blend13, v2); // 0.4-0.6区域使用材质3 result = lerp(result, blend14, v3); // 0.2-0.4区域使用材质4 result = lerp(result, shadowValue1, v4); // 0-0.2区域使用材质1 return result; }

GenshinToon_Face 核心实现

- 输入：BaseMap、SDF 距离场、ShadowMask、头部方向向量
- 关键算法：光源方向投影 → 左右侧判断 → SDF 硬边界采样
- 特色：有效解决面部"死亡光"问题，腮红使用 BaseMap Alpha 通道控制

核心代码片段（Face SDF Shadow）

half3 LpHeadHorizon = normalize(L - LpU); half value = acos(dot(LpHeadHorizon, headRightDir)) / PI; half sdf = step(mixValue, mixSdf); // 硬边界阴影 sdf = lerp(0, sdf, step(0, dot(LpHeadHorizon, headForwardDir)));

Vibecoding开发shader

ShiverToon.shader

概述

在标准 Toon 着色基础上，创新加入实时冷颤波浪变形效果的卡通角色 Shader。目标是让角色在特定场景下呈现出生动、自然的"受冷发抖"动态表现，同时保持 Toon 风格的美术可控性。

核心效果

- 分层冷颤系统：根据世界空间高度实现脚-腰-头三段不同振幅曲线
- 脉冲包络 + 随机失真：实现间歇性爆发抖动
- 动态几何描边：Inverted Hull 描边粗细随冷颤波浪相位实时变化
- 轮廓限制：仅在角色左右侧 Fresnel 边缘区域触发变形，避免面部扭曲

技术实现

- 顶点着色器中完成完整冷颤逻辑（Fresnel 检测 + 三段曲线 + 高频微颤）
- 描边 Pass 与主 Pass 共用波浪相位，保证一致性

项目意义
这是一个从"静态 Toon"向"动态风格化表现"探索的尝试，锻炼了我在顶点动画、程序化效果和 Shader 性能平衡方面的能力。

namespace清理工具

个人效率工具——解决Maya动画资产命名空间混乱的自动化方案

问题背景

在 Maya TA 工作中，频繁的资产交叉引用常导致命名空间混乱：多层嵌套、同名冲突、僵尸命名空间残留，严重影响场景加载和多人协作。

开发动机

离职后，我系统复盘了这些痛点，使用 Vibecoding 工作流独立开发了这款轻量、稳定的命名空间整理工具。

核心功能与技术亮点

- 智能扁平化：递归分析层级树，从深到浅合并父级命名空间
- 安全清理：通过 referenceQuery 精准区分引用/非引用，失败时回退 moveNamespace
- 智能重排序：根据引用文件 basename 自动编号 + 同步重命名引用节点（heroRN）
- 冲突解决：两阶段临时重命名策略（临时别名 → 最终目标名）
- 生产安全：自动备份场景 + undoInfo 原子块，一键 Ctrl+Z 回退

成果：将手动清理时间从 10+ 分钟缩短至 3 次点击，在本地复杂场景中稳定运行。

bug展示

- 在资产提交不规范的场景后，有时候会出现命名空间层级不正确、非正确引用资产导致命名空间错误、命名空间排序混乱等情况。

代码片段展示

1. find_referenced_namespaces —— 精准区分引用与非引用命名空间

for ns in user_namespaces: if ns in ref_namespaces: referenced.append(ns) else: nodes_in_ns = cmds.namespaceInfo(ns, listNamespace=True) or [] try: # 只要有一个节点被引用，整个命名空间就视为被引用 is_referenced = any( cmds.referenceQuery(node, isNodeReferenced=True) for node in nodes_in_ns ) except: is_referenced = False if is_referenced: referenced.append(ns) else: non_referenced.append(ns)

2. rename_namespace —— 安全重命名与冲突处理

def rename_namespace(old_ns, new_name, force_unique=True): if old_ns == new_name: return True abs_old = f':{old_ns}' if not old_ns.startswith(':') else old_ns target_name = new_name attempt = 0 while attempt < 20: try: abs_target = f':{target_name}' if not target_name.startswith(':') else target_name cmds.namespace(rename=(abs_old, target_name), force=True) return True except Exception as e: if "already exists" in str(e).lower(): if not force_unique: return False attempt += 1 target_name = f"{new_name}{attempt}" continue else: cmds.warning(f"重命名失败 {old_ns} -> {target_name}: {e}") return False return False

3. delete_unreferenced_namespaces —— 安全删除未引用命名空间

for ns in sorted(non_referenced, key=lambda x: x.count(':'), reverse=True): abs_ns = f':{ns}' if not ns.startswith(':') else ns try: # 主方案：直接合并到根 cmds.namespace(removeNamespace=abs_ns, mergeNamespaceWithRoot=True) except: # 备用方案：先移动到根，再删除 cmds.namespace(moveNamespace=[abs_ns, ":"], force=True) cmds.namespace(removeNamespace=abs_ns)

4. flatten_namespaces —— 嵌套命名空间扁平化

def flatten_namespaces(verbose=False): cmds.undoInfo(openChunk=True) try: while True: # 从深到浅处理 parents = [p for p in children_map if p in user_ns] parents.sort(key=lambda x: x.count(':'), reverse=True) for parent in parents: abs_parent = f':{parent}' if not parent.startswith(':') else parent cmds.namespace(removeNamespace=abs_parent, mergeNamespaceWithRoot=True) finally: cmds.undoInfo(closeChunk=True)

Plane-Cube Bump工具

Vibecoding 实践：AI 辅助开发的 Maya 实时顶点变形工具

背景

探索 AI 辅助 DCC 工具开发的实践项目。通过自然语言描述需求，快速生成并优化出一个实时顶点变形工具。

功能

在 Plane 上放置 Cube 后，当移动 Cube 时，Plane 会实时产生顶点推离变形，确保 Cube 不会穿透表面。核心基于有符号距离场计算。

技术亮点

- OpenMaya API（MFnMesh.getClosestPointAndNormal）
- QTimer 100ms 轮询 + UI 双向绑定
- 多轮安全校验迭代（_safety_push）

成果：实现了实时、稳定的几何交互效果。
